## 3. الوحدات الوظيفية (Functional Modules)

### 3.1 إدارة اللجان (Committee Management)

```
┌─────────────────────────────────────────────┐
│              إدارة اللجان                    │
├─────────────────────────────────────────────┤
│ • هيكل تنظيمي هرمي (مجالس ← لجان ← فرعية) │
│ • أنواع: دائمة / مؤقتة / مجلس / فريق       │
│ • إدارة الأعضاء (رئيس، أمين، عضو، مراقب)   │
│ • تتبع دورة الحياة (مسودة ← نشطة ← مغلقة)  │
│ • طلبات التعديل مع سير عمل الاعتماد         │
│ • مؤشرات أداء اللجنة (KPIs)                 │
│ • ربط المستندات والاجتماعات                 │
└─────────────────────────────────────────────┘
```

**أنواع اللجان:**
| النوع | الوصف |
|-------|-------|
| `permanent` | لجنة دائمة مستمرة بدون تاريخ انتهاء |
| `temporary` | لجنة مؤقتة لمهمة محددة |
| `council` | مجلس إداري أو استشاري |
| `main` | لجنة رئيسية تتبع لها لجان فرعية |
| `sub` | لجنة فرعية |
| `self_managed` | فريق ذاتي الإدارة |
| `cross_functional` | فريق متعدد التخصصات |

**حالات اللجنة:**
```
مسودة (Draft) → بانتظار الاعتماد (Pending Approval) → نشطة (Active)
                                                        ↓
                                               معلّقة (Suspended)
                                                        ↓
                                                مغلقة (Closed)
```

### 3.2 إدارة الاجتماعات (Meeting Management)

```
┌──────────────────────────────────────────────────────┐
│                  إدارة الاجتماعات                     │
├──────────────────────────────────────────────────────┤
│ • جدولة اجتماعات (حضوري / عن بُعد / مختلط)          │
│ • جدول أعمال ديناميكي مع مقدّمين وأوقات              │
│ • إدارة المدعوين (مطلوب / اختياري / منظّم / مقدّم)   │
│ • حجز قاعات الاجتماعات الفعلية                       │
│ • إنشاء اجتماعات Teams / Zoom تلقائياً               │
│ • تكامل تقويم Microsoft 365                          │
│ • غرفة اجتماع مباشرة داخل المنصة                    │
│ • إرسال إشعارات للمدعوين                             │
└──────────────────────────────────────────────────────┘
```

**أنواع الاجتماعات:**
| النوع | الوصف |
|-------|-------|
| `in_person` | حضوري — في قاعة اجتماعات فعلية |
| `online` | عن بُعد — عبر Teams أو Zoom |
| `hybrid` | مختلط — حضوري + عن بُعد |

**حالات الاجتماع:**
```
مسودة (Draft) → مجدول (Scheduled) → جاري (In Progress) → مكتمل (Completed)
                       ↓
                 ملغي (Cancelled)
```

### 3.3 محاضر الاجتماعات (Meeting Minutes — MOM)

```
┌──────────────────────────────────────────────────────┐
│               محاضر الاجتماعات                        │
├──────────────────────────────────────────────────────┤
│ • تسجيل الحضور والغياب مع الأسباب                   │
│ • ملاحظات ومناقشات لكل بند في جدول الأعمال           │
│ • تسجيل القرارات المتخذة                             │
│ • التوصيات وتحويلها لمهام تلقائياً                    │
│ • سير عمل الاعتماد (إرسال ← اعتماد/رفض)             │
│ • تصدير المحضر إلى Word / PDF ثنائي اللغة            │
│ • سجل كامل لتاريخ الإجراءات                         │
└──────────────────────────────────────────────────────┘
```

**حالات الحضور:**
| الحالة | الوصف |
|--------|-------|
| `present` | حاضر |
| `absent` | غائب |
| `excused` | معتذر |
| `late` | متأخر |

**أسباب الغياب:** إجازة مرضية، سفر رسمي، إجازة شخصية، تعارض مواعيد، بدون إشعار، أخرى

**حالات المحضر:**
```
مسودة (Draft) → بانتظار الاعتماد (Pending Approval) → معتمد (Approved)
                                                       ↓
                                                مرفوض (Rejected) → مسودة
```

### 3.4 إدارة المهام (Task Management)

```
┌──────────────────────────────────────────────────────┐
│                   إدارة المهام                        │
├──────────────────────────────────────────────────────┤
│ • إنشاء مهام من توصيات المحاضر تلقائياً              │
│ • تعيين المسؤول + تاريخ الاستحقاق + الأولوية         │
│ • تتبع التقدم (0% ← 100%)                           │
│ • دعم المهام الفرعية مع حساب تلقائي للتقدم           │
│ • تصنيف المهام (فئة + مستفيد)                        │
│ • رقم إيصال للتتبع الرسمي                            │
│ • تصفية حسب: الحالة، الأولوية، المسند إليه            │
└──────────────────────────────────────────────────────┘
```

**الأولويات:**
| المستوى | اللون |
|---------|-------|
| `low` منخفضة | أخضر |
| `medium` متوسطة | أصفر |
| `high` عالية | برتقالي |
| `critical` حرجة | أحمر |

**حالات المهمة:**
```
معلّقة (Pending) → جارية (In Progress) → مكتملة (Completed)
                                          ↓
                                   ملغاة (Cancelled)
     ↓
متأخرة (Overdue) ← [تلقائي عند تجاوز تاريخ الاستحقاق]
```

1. **ثنائية اللغة الكاملة** — كل حقل في قاعدة البيانات له نسختين (عربي/إنجليزي) مع دعم RTL
2. **لوحة معلومات قابلة للتخصيص** — سحب وإفلات مع حفظ التخطيط لكل مستخدم
3. **محرك سير عمل مرن** — آلة حالات JSON قابلة لأي سيناريو اعتماد
4. **استبيان مباشر تفاعلي** — بمسح QR Code ونتائج فورية
5. **تكامل Microsoft 365** — Teams + Calendar + Azure AD
6. **نظام إشعارات متعدد القنوات** — داخلي + بريد + SMS + Push
7. **تصعيد SLA تلقائي** — تنبيهات عند تجاوز المهل الزمنية
8. **تقييم أداء اللجان** — بقوالب ومعايير وأوزان قابلة للتخصيص
9. **سحب عشوائي للفائزين** — تحفيز المشاركة في الاستبيانات
10. **تطبيق جوال كامل** — مع دعم offline ومصادقة بيومترية
11. **تصنيف أمني للملفات** — 4 مستويات (عام ← مقيّد)
12. **سجل تدقيق شامل** — تسجيل كل عملية بكتابة مؤجلة عالية الأداء

## 15. النشر على Windows Server – دليل سريع

### الخطوات الأساسية

1. **تثبيت المتطلبات**: IIS + WebSockets + URL Rewrite + .NET 8 Hosting Bundle + Node.js 20 + PostgreSQL 16 + Redis + MinIO
2. **تعديل البروفايل**: `deploy\DEPLOYMENT-PROFILE.ps1` — بيانات الاتصال والأسرار
3. **تشغيل النشر**: `.\Deploy-UohPlatform.ps1` (كمسؤول PowerShell)
4. **التحقق**: `.\Deploy-UohPlatform.ps1 -ValidateOnly`

### عناوين الخدمات بعد النشر

| الخدمة | العنوان |
|--------|---------|
| الواجهة الأمامية | `https://meetings.uoh.edu.sa` |
| واجهة API | `https://api.meetings.uoh.edu.sa` |
| فحص الصحة | `https://api.meetings.uoh.edu.sa/health` |
| Swagger (تطوير فقط) | `https://api.meetings.uoh.edu.sa/swagger` |

### أوامر الصيانة

```powershell
# إعادة تشغيل API
Restart-WebAppPool -Name "UOH-API-Pool"

# إعادة تشغيل الواجهة
Restart-WebAppPool -Name "UOH-Web-Pool"

# فحص صحة API
Invoke-RestMethod https://api.meetings.uoh.edu.sa/health

# نسخ احتياطي يدوي
& "C:\Program Files\PostgreSQL\16\bin\pg_dump" -h localhost -U uoh -d uoh_meetings -Fc -f "C:\backups\manual.dump"

# استعادة من نسخة احتياطية
& "C:\Program Files\PostgreSQL\16\bin\pg_restore" -h localhost -U uoh -d uoh_meetings --clean "C:\backups\manual.dump"
```

## 16. نشر تطبيق الجوال (Mobile Deployment)

### EAS Build

```bash
# تثبيت EAS CLI
npm install -g eas-cli

# تسجيل الدخول
eas login

# بناء Android (AAB للمتجر)
cd apps/mobile
eas build --platform android --profile production

# بناء iOS
eas build --platform ios --profile production

# رفع للمتاجر
eas submit --platform android --profile production
eas submit --platform ios --profile production
```
