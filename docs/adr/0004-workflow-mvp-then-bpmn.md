# ADR 0004: Workflow approach (MVP then BPMN)

## Status

Accepted

## Context

المتطلبات تشمل محرك سير عمل ديناميكي بواجهة مرئية. هذا يحتاج وقتًا كبيرًا وقد يعرقل إطلاق النسخة الأولى لو تم تنفيذه كاملًا منذ البداية.

## Decision

- **المرحلة الأولى (MVP)**: محرك Workflow مخصص يغطي:
  - حالات/انتقالات (State machine)
  - خطوات اعتماد (Approvals) + أدوار/موافقين
  - شروط بسيطة (Rules) + تصعيدات
  - إشعارات Email/Push
- **المرحلة الثانية**: تقييم اعتماد BPMN (مثل Camunda) للـ builder المرئي إذا ثبتت الحاجة لتصميمات معقدة متعددة.

## Consequences

- إطلاق أسرع مع تغطية 80% من الاحتياج.
- قد يتطلب migration لاحقًا إن تم تبني BPMN لمحاكاة تدفقات المرحلة الأولى.
